# Пирожковый Диспетчер

Пирожковый Диспетчер, или Sirius Plus, помогает следить за расписанием `my.sirius.online`, получать напоминания и автоматически записываться на выбранные события в момент открытия регистрации.

Сайт: [sirius.rusanoff.ru](https://sirius.rusanoff.ru/)<br>
Telegram-канал: [@sirius_pirozhki](https://t.me/sirius_pirozhki)

> Проект не является официальным продуктом Sirius. Он работает с пользовательским аккаунтом только после явного входа владельца.

## Возможности

- Просмотр расписания, поиска, фильтров и статусов записи.
- Автозапись на события при открытии регистрации с выбором приоритета.
- Три приоритета автозаписи: быстрый, обычный и медленный. Стоимость и частота зависят от выбранного приоритета и уровня доверия аккаунта.
- Журнал попыток снайпера: видно, когда и как сервис пытался записать пользователя.
- Уведомления об открытии регистрации, изменениях расписания, напоминания о событиях и ответы администрации.
- Web Push-уведомления для браузера и Android-оболочка с просмотром ранее открытых страниц без сети.
- Собственные события, напоминания, расписание трансферов и отображение номера автобуса.
- Сириус Коины: резервирование за автозапись, переводы, промокоды и реферальная программа.
- Сириус Полимаркет с рынками, создаваемыми администраторами.
- Обращения в поддержку: пользователь видит историю диалога, администратор может отвечать и скрывать обращения только у себя.
- Админ-панель для управления пользователями, уровнем доверия, администраторами, промокодами, обращениями и рынками.
- Changelog, который показывается один раз после обновления сайта.

## Как работает автозапись

Пользователь добавляет событие в слежку и выбирает приоритет. Сервис резервирует нужное число Сириус Коинов, заранее запускает задачу и повторяет попытку записи до успеха или окончания окна записи. При успешной записи резерв списывается; при отмене слежки или неудаче возвращается.

Опрос расписания не выполняется одинаково для всех аккаунтов каждую минуту. Интервал зависит от уровня доверия и содержит небольшую случайную погрешность, чтобы запросы не выглядели как синхронный бот-трафик. На странице событий есть ручное обновление.

## Уведомления

В браузере доступны системные уведомления и звуковое напоминание, пока сайт открыт. В Android-приложении есть системный канал уведомлений со звуком при работающем приложении.

Фоновые уведомления для закрытого Android-приложения реализованы через Firebase Cloud Messaging. Для включения нужны закрытые конфигурационные файлы Firebase для APK и сервера; инструкция находится в [docs/android-fcm.md](docs/android-fcm.md). WebView сам по себе не может надёжно работать в фоне бесконечно.

## Android-приложение

Android-оболочка открывает сайт, сохраняет ранее посещённые страницы и показывает их в офлайн-режиме. Доступность определяется запросом к самому сайту, а не наличием Wi-Fi: белые списки и неработающая сеть тоже переводят приложение в офлайн-режим.

В офлайне доступны только заранее сохранённые страницы. Запись, обновление данных, профиль, переводы и другие сетевые действия блокируются. После установки нужно открыть нужные страницы хотя бы один раз с доступным сайтом.

Скачать APK: [GitHub Releases](https://github.com/sonofstrange/sirius-plus/releases).

Для сборки из исходников открой папку `android` в Android Studio. Нужны встроенный JDK 21, Android SDK Platform 36 и Gradle Wrapper 8.10.2. Debug APK создаётся командой:

```powershell
cd android
./gradlew.bat assembleDebug
```

Готовый файл: `android/app/build/outputs/apk/debug/app-debug.apk`.

## Безопасность и приватность

- Вход по одноразовому коду Sirius запрашивает код на email в отдельном чистом контексте Playwright. Для повторного запроса кода хранится только зашифрованный email, без пароля.
- После входа сервис получает токен Sirius; одноразовый код не сохраняется и не передаётся сторонним сервисам.
- Токены и сохранённые данные входа шифруются Fernet-ключом в SQLite.
- Сессия сайта хранится в HttpOnly-cookie, а права администратора проверяются по ранее верифицированному токену, а не по произвольному JWT payload.
- Для обращений, переводов, промокодов и админ-действий требуется авторизованная сессия.
- В проекте нет сторонней аналитики и нет отправки данных аккаунта в рекламные системы.
- Ключи, база данных, VAPID-ключ и runtime-файлы исключены из Git через `.gitignore`.

Публичный код создан для аудита и изучения. Официальный экземпляр дополнительно привязан к приватному persistent data volume через файл активации. Это не замена юридической или инфраструктурной безопасности, но не даёт скопированному репозиторию автоматически стать рабочим экземпляром сайта.

## Технологии

- Python 3.12, FastAPI, Uvicorn.
- Jinja2 SSR, ванильные JavaScript и CSS.
- Playwright + Chromium для взаимодействия с Sirius.
- SQLite для сессий, токенов, кеша, слежки, уведомлений и коинов.
- Docker Compose для production.
- Android WebView, Java и Gradle для APK.

## Структура

```text
main.py              FastAPI, страницы и HTTP API
sirius_api.py        авторизация и запросы Sirius через Playwright
poller.py            опрос расписания, напоминания и автозапись
storage.py            SQLite и шифрование токенов
templates/           SSR-шаблоны
static/              стили, клиентский JavaScript, Service Worker
android/             Android-оболочка и инструкции по сборке
docs/                документация для интеграций
nginx/               конфигурация и страница обслуживания
scripts/             служебные скрипты для ключей и упаковки
```

## Запуск официального экземпляра

Production использует Docker Compose. Контейнер слушает только `127.0.0.1:8000`; внешний HTTPS и статика обслуживаются Nginx. Данные находятся в отдельном persistent volume, поэтому пересборка контейнера не стирает базу, ключ шифрования и push-ключ.

Основные переменные окружения:

| Переменная | Назначение |
| --- | --- |
| `DB_PATH` | путь к SQLite-базе |
| `SIRIUS_HOST` | хост Uvicorn |
| `SIRIUS_PORT` | порт Uvicorn |
| `TOKEN_ENCRYPTION_KEY` | Fernet-ключ, если не используется файл ключа в data volume |
| `WEB_PUSH_SUBJECT` | контакт для VAPID Web Push |
| `FCM_SERVICE_ACCOUNT_FILE` | путь к JSON-ключу Firebase для отправки Android Push |
| `DRONEBET_OUTBOUND_TOKEN` | токен DroneBet для запросов Sirius Plus к DroneBet |
| `DRONEBET_INBOUND_TOKEN` | отдельный токен Sirius Plus для запросов DroneBet к Sirius Plus |

На первом развёртывании создаётся ключ шифрования. Файл активации официального экземпляра создаётся отдельно после сборки:

```bash
docker compose run --rm sirius-plus python scripts/create_instance_seal.py
```

## Партнёрство с DroneBet

В разделе Сириус Коинов есть вкладка DroneBet. Обмен работает по фиксированному курсу:

```text
1000 печенек DroneBet = 1 Сириус Коин
```

Собственный рынок DroneBet удалён. Во вкладке `Сириус Коины -> DroneBet` можно
привязать аккаунт одноразовым кодом и обменять валюту по курсу `1 000` печенек
за `1` Сириус Коин. Все переводы серверные, идемпотентные и не передают Sirius
токен или пароль партнёру. Спецификация: [docs/dronebet-api.md](docs/dronebet-api.md).

## Защита от перегрузки

Nginx ограничивает общий поток запросов, число соединений с одного IP и отдельно
партнёрский API. Перед заменой server-конфига установите файл зон лимитов в
`/etc/nginx/conf.d/sirius-plus-limits.conf`, затем проверьте конфигурацию через
`nginx -t`. Это не заменяет защиту дата-центра от большого DDoS-канала, но
отсекает обычный HTTP-флуд и медленные соединения до Python-приложения.
