# Интеграция Sirius Plus и DroneBet

Статус: согласование API. Публичные маршруты для обмена ещё не включены.

## Назначение

Сервис позволяет обменивать баланс между двумя системами по фиксированному курсу:

```
2000 печенек DroneBet = 1 Сириус Коин
```

Обмен доступен в обе стороны. Браузер пользователя не должен обращаться к API партнёра напрямую: все запросы выполняются сервером Sirius Plus.

## Идентификация пользователя

Перед первой операцией пользователь привязывает DroneBet-аккаунт через одноразовый код или OAuth. В обменных запросах используется только стабильный `dronebet_user_id` партнёра. Sirius Plus не передаёт пароль, Sirius JWT, email или имя пользователя.

## Авторизация запросов

Каждый запрос Sirius Plus к DroneBet подписывается общим секретом `DRONEBET_PARTNER_SECRET`.

```text
X-Partner-Id: sirius-plus
X-Timestamp: 1760000000
X-Request-Id: 66f25b85-4eb6-4cae-83a7-9fe0b6e9f8e7
X-Signature: <hex HMAC-SHA256>
```

Строка для подписи:

```text
<HTTP method>\n<path>\n<X-Timestamp>\n<raw JSON body>
```

`X-Timestamp` действителен не более 60 секунд. `X-Request-Id` должен быть UUID и храниться партнёром минимум 24 часа: повтор того же запроса возвращает исходный результат, а не повторно меняет баланс.

## Предлагаемые маршруты DroneBet

Базовый URL: `https://<dronebet-domain>/api/v1/partners/sirius-plus`.

### Создать подготовленную операцию

`POST /operations`

```json
{
  "operation_id": "uuid",
  "dronebet_user_id": "partner-user-123",
  "direction": "cookies_to_coins",
  "cookies": 2000,
  "sirius_coins": 1
}
```

Допустимые `direction`: `cookies_to_coins` и `coins_to_cookies`. Суммы всегда целые, положительные и соответствуют курсу.

Для `cookies_to_coins` DroneBet проверяет и удерживает печеньки. Для `coins_to_cookies` операция только подготавливается: Сириус Коин остаётся зарезервированным в Sirius Plus до успешного завершения.

Ответ `201`:

```json
{
  "operation_id": "uuid",
  "status": "prepared",
  "expires_at": "2026-07-14T12:05:00Z"
}
```

При нехватке печенек возвращается `409` с `{ "code": "insufficient_cookies" }`.

### Подтвердить операцию

`POST /operations/{operation_id}/commit`

Для `cookies_to_coins`: DroneBet окончательно списывает удержанные печеньки; Sirius Plus начисляет Сириус Коины только после ответа `completed`.

Для `coins_to_cookies`: Sirius Plus сначала резервирует Сириус Коин, затем DroneBet начисляет печеньки и возвращает `completed`; после этого Sirius Plus окончательно списывает резерв.

```json
{
  "operation_id": "uuid",
  "status": "completed",
  "cookies_delta": -2000,
  "cookies_balance": 18450
}
```

Повторный `commit` с тем же `operation_id` обязан вернуть тот же результат.

### Отменить операцию

`POST /operations/{operation_id}/cancel`

Освобождает удержанные печеньки, если статус ещё `prepared`. Sirius Plus одновременно снимает свой резерв. Отмена завершённой операции возвращает `409` с `{ "code": "already_completed" }`.

### Получить статус

`GET /operations/{operation_id}`

Используется для восстановления после таймаута или перезапуска. Возможные статусы: `prepared`, `completed`, `cancelled`, `expired`.

## Восстановление после ошибок

Таймаут не означает неудачу. Sirius Plus сначала вызывает `GET /operations/{operation_id}`. Если партнёр отвечает `completed`, локальная операция завершается; если `prepared`, выполняется повторный `commit` с тем же идентификатором; если `cancelled` или `expired`, локальный резерв снимается.

Нельзя использовать случайный новый `operation_id` при повторе. Это единственная защита от двойного начисления при проблемах сети.

## Ошибки

Все ошибки возвращаются JSON-объектом:

```json
{ "code": "invalid_signature", "message": "Signature verification failed" }
```

Ожидаемые коды: `400` для неверного тела, `401` для подписи, `403` для отключённого партнёра, `404` для неизвестной операции, `409` для конфликта состояния или недостатка баланса, `429` для rate limit и `500` для временной ошибки.

## Что нужно согласовать перед включением

1. Базовый URL и идентификатор партнёра.
2. Формат привязки аккаунтов.
3. Способ передачи и ротации общего секрета.
4. Максимум печенек и коинов за одну операцию, а также rate limit.
5. Тестовый стенд с отдельными балансами.
